# 코드 리뷰 체크리스트

## 정확성
- 비즈니스 로직이 의도대로 동작하는가
- 엣지 케이스를 처리하고 있는가
- 에러 핸들링이 적절한가
- Null/undefined 체크
- Off-by-one 에러

## 코드 품질
- 함수가 20-50줄 이내인가
- Type hints가 모든 함수에 있는가
- 네이밍이 의도를 드러내는가
- 코드 중복이 없는가
- 단일 책임 원칙을 따르는가

## 보안
- 민감 정보(API 키, 비밀번호)가 하드코딩되어 있지 않은가
- SQL 인젝션, XSS 등 보안 취약점이 없는가
- 사용자 입력이 적절히 검증되는가
- 인증/인가 처리가 올바른가

## 성능
- N+1 쿼리 패턴이 없는가
- 불필요한 루프나 중복 연산이 없는가
- 대용량 데이터 처리 시 메모리 사용이 적절한가
- 알고리즘 복잡도가 적절한가

## 테스트
- 변경사항에 대한 테스트가 있는가
- 기존 테스트가 깨지지 않는가
- 엣지 케이스 테스트 포함
- 비동기 테스트 패턴 올바른가 (pytest-asyncio, tokio::test 등)

## 출력 등급
- 🔴 **Critical**: 반드시 수정 필요
- 🟡 **Warning**: 수정 권장
- 🟢 **Suggestion**: 개선 제안
- 각 항목에 파일명:라인넘버와 구체적 수정 방법 포함
